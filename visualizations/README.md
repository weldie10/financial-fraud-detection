# Visualizations Directory

This directory contains visualization outputs generated by the EDA module during the preprocessing pipeline.

## Generating Visualizations

### Option 1: Automatic Generation (Recommended)

Visualizations are automatically created when using the preprocessing pipeline with `perform_eda=True`:

```python
from src.preprocessor import PreprocessingPipeline

pipeline = PreprocessingPipeline(
    data_dir="data/raw",
    output_dir="data/processed"
)

# Process with EDA enabled - this generates visualizations
processed_df, metadata = pipeline.process_fraud_data(
    fraud_data_file="Fraud_Data.csv",
    perform_eda=True,  # This generates visualizations
    handle_imbalance=True
)
```

Visualizations will be saved to `data/processed/eda_outputs/` directory.

### Option 2: Generate Sample Visualizations

To generate sample visualizations for the report (without running the full pipeline):

```bash
# Activate virtual environment
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Run the visualization generation script
python scripts/generate_visualizations.py
```

This will create sample visualizations in `data/processed/eda_outputs/` that match the report structure.

## Viewing Visualizations

### In Markdown Viewers

- **GitHub/GitLab**: Visualizations will display automatically if images are in the repository
- **VS Code**: Install "Markdown Preview Enhanced" extension for best viewing
- **Jupyter**: Visualizations can be embedded directly in notebooks

### Direct Viewing

Open the PNG files directly:
```bash
# Linux/Mac
open data/processed/eda_outputs/class_distribution.png

# Windows
start data/processed/eda_outputs/class_distribution.png
```

## Troubleshooting

**Visualizations not showing in report?**

1. Ensure visualizations have been generated (run the script or pipeline)
2. Check that files exist: `ls data/processed/eda_outputs/*.png`
3. Verify image paths in the report match actual file locations
4. For GitHub/GitLab: Ensure images are committed to the repository

**Script fails with import errors?**

1. Activate your virtual environment: `source venv/bin/activate`
2. Install dependencies: `pip install -r requirements.txt`
3. Verify installation: `python -c "import matplotlib; import seaborn"`
